## A.1 Overview

The My Recovery App (MYRA) is a purpose-built Android passive sensing application developed to support the present research study. MYRA operates silently in the background of participants' smartphones, collecting continuous behavioural and physiological sensor data without requiring active participant input beyond initial setup and informed consent.

Data collected by MYRA is structured, validated, and transmitted to the REDCap research data platform, where it is consumed by the SHARON (Supporting Human Addiction Recovery through Observational Networks) machine learning pipeline for the generation of dropout and relapse predictions.

## A.7 Quality Assurance — Healthcheck Script

An automated healthcheck script (`mrt_health_gate.sh`) was developed to validate the operational state of all MYRA components. The script evaluates:

- Service running status
- Worker output file presence and recency
- Permission grant status
- Upload cadence and success rate
- WorkManager job queue state
- Schema integrity

**Dry-run result at build lock:**

| Metric | Result |
|---|---|
| Passes | 80 |
| Failures | 0 |
| Warnings | 2 (non-actionable) |

### A.7.1 Non-Actionable Warnings

Two warnings were present at build lock. Both were investigated, understood, and confirmed to require no remediation:

**Warning 1 — Historical service restarts:**
Eight service restarts were recorded 14 days prior to the healthcheck. These occurred during the active development and bug-fixing phase, before the stable build was achieved. They are not indicative of instability in the locked build.

**Warning 2 — Health snapshot timing mismatch:**
The `daily_health.csv` health snapshot recorded a `none` upload status during one healthcheck cycle. This is a transient timing artefact caused by the healthcheck running between write cycles of the health snapshot worker. It does not reflect an actual upload failure.

## A.8 SHARON Integration

MYRA is the data collection instrument for the SHARON machine learning model. SHARON consumes structured daily metrics uploaded via REDCap and produces the following outputs:

- Dropout prediction probability (range 0–1)
- Dropout prediction calibrated score
- Dropout prediction confidence flag
- Relapse prediction probability (range 0–1)
- Relapse prediction calibrated score
- Relapse prediction confidence flag
- Dropout trajectory label
- Clinician narrative report
- SHAP feature importance values

**SHARON model version at P001 enrolment:** `sharon_v20.8.pkl`
